# 관리자 운영 플레이북 (일일 루틴)

## 목적
- 운영 통합 센터(`/admin/operations`)에서 주문/대여/교체신청을 **우선순위 기반으로 매일 처리**하기 위한 실무 루틴입니다.
- 팀 내 동일한 처리 순서를 유지해 누락과 중복 처리를 줄입니다.

## 권장 시작 시간대
- 오전 1회(업무 시작 직후)
- 오후 1회(정산/배송 반영 전)
- 마감 전 1회(미처리 건 재점검)

## 일일 처리 순서
1. **오늘 해야 할 일 수치 확인**
   - 상단 요약의 긴급/주의/미처리 건수를 확인합니다.
   - 전일 대비 급증한 항목이 있으면 먼저 원인을 확인합니다.

2. **프리셋: 결제불일치 확인**
   - `결제불일치 확인` 프리셋으로 주의(데이터 연결 오류) 건을 우선 조회합니다.
   - 운영함에서 `검수필요`는 오류가 아닌 정책 확인 신호이므로, 주의 처리 후 순차 점검합니다.
   - 조치 예: 잘못 연결된 신청서 분리/재연결, 결제 상태 동기화 점검.

3. **프리셋: 통합건 검수**
   - `통합건 검수` 프리셋으로 연결된 주문/대여 + 신청서를 점검합니다.
   - 그룹 대표 문서와 하위 문서의 상태 라벨이 일치하는지 확인합니다.
   - 조치 예: 상태 전이 누락(신청서만 완료/주문 미완료) 보정.

4. **프리셋: 단독 신청서 처리**
   - `단독 신청서 처리` 프리셋으로 교체서비스 단독 신청만 확인합니다.
   - 고객 연락 필요 여부, 결제 완료 여부, 작업 배정 여부를 확인합니다.

5. **정산 화면 이동 및 마감 점검**
   - 필요한 건은 각 행의 정산 이동 링크 또는 상단 `정산 관리` 버튼으로 이동합니다.
   - 당일 처리 건 중 정산 반영 누락이 없는지 확인합니다.

6. **저장된 뷰 링크 공유**
   - 운영 통합 센터의 `현재 뷰 링크 복사` 버튼으로 URL을 복사합니다.
   - 슬랙/노션 등에 “오늘 점검 기준 뷰”를 공유해 팀 동일 기준을 맞춥니다.

## 인수인계 메시지 템플릿
- 오늘 우선 점검 뷰: (복사한 링크)
- 미해결 긴급 건: n건
- 미해결 주의 건(연결 오류): n건
- 미해결 검수필요 건: n건
- 요청사항: (예: 결제 동기화 확인 필요, 고객 응대 필요)
