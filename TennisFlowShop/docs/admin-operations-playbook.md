# 관리자 운영 플레이북 (일일 루틴)

## 목적
- 운영 통합 센터(`/admin/operations`)에서 주문/대여/교체신청을 **우선순위 기반으로 매일 처리**하기 위한 실무 루틴입니다.
- 팀 내 동일한 처리 순서를 유지해 누락과 중복 처리를 줄입니다.

## 권장 시작 시간대
- 오전 1회(업무 시작 직후)
- 오후 1회(정산/배송 반영 전)
- 마감 전 1회(미처리 건 재점검)

## 일일 처리 순서
1. **오늘 해야 할 일 수치 확인**
   - 상단 요약의 긴급/주의/미처리 건수는 **현재 필터 결과 기준**으로 계산됩니다.
   - 전일 대비 급증한 항목이 있으면 먼저 원인을 확인합니다.

2. **프리셋: 주의(오류) 우선 점검**
   - `주의(오류) 우선 점검` 프리셋으로 주의(데이터 연결 오류) 건을 우선 조회합니다.
   - 운영함에서 `검수필요`는 오류가 아닌 정책 확인 신호이므로, 배지의 `검수 사유`를 확인해 주의 처리 후 순차 점검합니다.
   - 조치 예: 잘못 연결된 신청서 분리/재연결, 결제 상태(용어 표준: 결제완료/결제대기) 동기화 점검.

3. **프리셋: 통합건 검수 + 검수필요 필터**
   - `통합건 검수` 프리셋으로 연결된 주문/대여 + 신청서를 점검합니다.
   - 필요 시 위험 신호 필터를 `검수필요만`으로 바꿔 정책 확인 대상만 빠르게 재검토합니다.
   - 그룹 대표 문서와 하위 문서의 상태 라벨이 일치하는지 확인합니다.
   - 조치 예: 상태 전이 누락(신청서만 완료/주문 미완료) 보정.

4. **프리셋: 단독 신청서 처리 + 결제 라벨 확인**
   - `단독 신청서 처리` 프리셋으로 교체서비스 단독 신청만 확인합니다.
   - 고객 연락 필요 여부, 결제 완료 여부, 작업 배정 여부를 확인합니다.
   - 신청서 결제 라벨 `패키지차감 / 주문결제포함 / 대여결제포함 / 확인필요`는 파생 결과이므로 상세 맥락(연결 주문/대여/패키지 적용 여부)과 함께 판단합니다.

5. **금액 해석 규칙으로 0원 오해 방지**
   - 메인 금액(굵은 숫자)은 항상 실제 청구/결제 기준 금액입니다.
   - 신청서 메인 금액이 `0원`이면 보조 문맥(`주문결제포함`, `대여결제포함`, `패키지차감`, `별도청구없음`, `확인필요`)을 함께 확인합니다.
   - `기준금액`(예: `serviceFeeBefore`)은 보조 정보이며, 실제 청구 금액으로 오해하지 않도록 메인 금액과 분리해 봅니다.
   - 대표 패턴
     - 단독 유료 신청서: 메인 금액이 0원이 아님
     - 주문 포함 신청서: `0원 · 주문결제포함`
     - 대여 포함 신청서(Flow 7): `0원 · 대여결제포함`
     - 패키지 차감 신청서: `0원 · 패키지차감`

6. **정산 화면 이동 및 마감 점검**
   - 필요한 건은 각 행의 정산 이동 링크 또는 상단 `정산 관리` 버튼으로 이동합니다.
   - 당일 처리 건 중 정산 반영 누락이 없는지 확인합니다.

7. **저장된 뷰 링크 공유**
   - 운영 통합 센터의 `현재 뷰 링크 복사` 버튼으로 URL을 복사합니다. (`warnFilter`, `warnSort` 포함)
   - 슬랙/노션 등에 “오늘 점검 기준 뷰”를 공유해 팀 동일 기준을 맞춥니다.
   - `주의(오류)만 보기`를 켠 상태에서는 위험 신호 필터가 자동으로 `주의만`으로 고정되어 모순 조합을 방지합니다.


## 공통 상태 읽기 규칙 (주문/대여/신청 공통)

- 관리자 화면에서 상태는 아래 순서로 읽습니다.
  1) **업무 상태**: 주문 상태 / 대여 상태 / 신청 상태
  2) **결제 상태(용어 표준: 결제완료/결제대기)**: 결제완료 / 결제대기 / 결제취소 등
  3) **배송/출고 상태**: 운송장 등록 여부, 출고/반납 진행 여부
- 대여 문서처럼 결제 필드가 비어 있을 수 있는 케이스는 `대여 상태/paidAt` 기반 파생값으로 `결제완료` 또는 `결제대기`를 표시합니다.
- `-` 표시는 원칙적으로 지양하고, 불가피하게 비어 있는 경우 `결제정보 없음(문서 미기입)`처럼 이유를 함께 표기합니다.
- Flow 7(대여+신청) 검수 시에는 문서를 분리해서 읽습니다.
  - 대여: 업무 상태(대여중/반납완료 등) + 결제 상태(용어 표준: 결제완료/결제대기)
  - 신청서: 작업 상태(접수완료/작업 중/교체완료 등) + 결제 문맥(대여결제포함/패키지차감 등)

## 인수인계 메시지 템플릿
- 오늘 우선 점검 뷰: (복사한 링크)
- 미해결 긴급 건: n건
- 미해결 주의 건(연결 오류): n건
- 미해결 검수필요 건: n건
- 요청사항: (예: 결제 동기화 확인 필요, 고객 응대 필요)

## 연결 업무 다음 할 일 가이드 (2026-02)

### 1) 주문 + 신청
- 현재 단계 판단:
  - 주문 결제 전이면 **결제 확인 단계**
  - 신청서가 `검토 중/접수완료`이면 **신청 접수 단계**
  - 신청서가 `작업 중`이면 **교체 작업 단계**
- 대표 다음 할 일:
  - `주문 결제 확인 및 신청 작업 가능 상태 점검`
  - `신청서 작업 상태 확인 필요`
  - `작업 완료 후 신청서 교체완료 처리 필요`
- 화면 반영 위치:
  - 운영 통합 센터 그룹 행/확장에서 `현재 업무 단계 + 다음 할 일`을 우선 확인
  - 주문 상세(`/admin/orders/[id]`)의 `연결 업무 가이드` 카드에서 동일 문맥으로 재확인

### 2) 대여 + 신청 (Flow 7)
- 현재 단계 판단:
  - 대여 결제 전: **대여 결제 확인 단계**
  - 결제완료 + 출고 운송장 미등록: **출고 준비 단계**
  - 신청서 `작업 중`: **교체 작업 단계**
  - 신청서 `교체완료` + 대여 아직 시작 전: **인도 대기 단계**
- 대표 다음 할 일:
  - `대여 결제 확인 필요`
  - `출고 정보 등록 필요`
  - `신청서 교체완료 처리 필요`
  - `사용자가 수령하면 대여 시작 처리 필요`
- 화면 반영 위치:
  - 운영 통합 센터의 검수 사유 옆 `다음 할 일` 문구
  - 대여 상세(`/admin/rentals/[id]`)의 `연결 업무 가이드` 카드
  - 신청서 상세(`/admin/applications/stringing/[id]`)의 `연결 업무 기준 관리자 할 일` 카드

### 3) 결제 문맥 케이스(패키지/주문결제포함/대여결제포함)
- 신청서 결제 라벨이 아래 중 하나면 금액 0원이라도 **오류가 아닐 수 있음**.
  - `패키지차감`
  - `주문결제포함`
  - `대여결제포함`
- 운영자가 반드시 볼 체크포인트:
  - 결제 라벨과 연결 문서(kind/orderId/rentalId)가 일치하는지
  - 0원 신청서의 `기준금액`은 참고값이며, 실제 청구액이 아니라는 점
  - 결제/상태 자동 동기화가 아니라 **안내 문구 기준으로 수동 처리 순서**를 지키는지
