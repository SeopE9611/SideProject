# 관리자 리스트 선택 상태 초기화 정책

관리자 목록 화면에서 체크박스 등 **행 선택 상태(selection state)** 는 아래 공통 규칙을 따른다.

## 공통 규칙

- 검색어/필터/정렬 조건이 변경되면 선택 상태를 즉시 초기화한다.
- 페이지 이동만 발생한 경우에는 화면 정책에 따라 유지하거나 초기화할 수 있으나, 기본값은 **유지하지 않고 초기화**를 권장한다.
- 대량 작업(일괄 비활성화/삭제 등) 성공 후에는 선택 상태를 초기화한다.

## 적용 이유

- 필터 변경 전후의 데이터 집합이 달라져, 이전 선택이 의도치 않은 대상을 가리킬 수 있다.
- 운영자가 다른 조건으로 전환하는 순간 선택 상태가 남아 있으면 오작동 위험이 커진다.

## 구현 기준

- 쿼리 상태 훅(`useAdminListQueryState`)으로 검색/필터/정렬 변경 시 페이지를 1로 리셋한다.
- 선택 상태는 검색/필터/정렬 상태값을 `useEffect` 의 의존성으로 두고, 변경 시 빈 배열로 초기화한다.
